# M1.1 晨报可用化（手动编辑 + 发送）

目标：在本地可手动维护晨报内容，并可一键发送；同时预留后续移动端交互接口。

## 新增能力
- 晨报草稿本地化存储（`data/morning_brief.md`）
- API 支持读取/保存晨报内容
- 手动发送接口（可调用 notifier）
- 本地发送通道（写入 `data/outbox/`），用于无 webhook 的场景

## 接口
- `GET /reports/morning-brief`：获取当前晨报（草稿或默认模板）
- `POST /reports/morning-brief`：保存晨报草稿
- `POST /reports/morning-brief/send`：立即发送当前晨报

## 出口（可验证）
- 手动保存晨报后，`GET` 返回最新内容
- 调用 `send` 后在 outbox 产生文件或成功推送到企业微信
- `scripts/send_morning_brief.ps1` 可一键推送

## 依赖安装备注
- 若 pip 受代理/SSL 影响，可先设置：`$env:NO_PROXY='*'`
