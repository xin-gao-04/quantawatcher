# M0 规格冻结（v0.1）

目标：在 2026-01-28 冻结第一阶段最小闭环规格，确保进入实现前的统一共识与可执行清单。

## 固定默认值（若后续用户确认，将覆盖）
- 盘中采样粒度：60 秒
- 首个推送通道：企业微信 webhook
- 指标优先级：板块热度优先
- 数据源约束：默认免费源优先，稳定性与合规优先于完备性

## MVP 交付定义（必须全部满足）
1) 常驻运行 + 崩溃自恢复
2) 每日晨报生成并推送
3) 盘中轮询 watchlist，至少产生两类事件
4) 每条推送对应持久化 Event 记录，包含触发解释
5) Web 控制台可编辑 watchlist 与阈值

## 事件类型（v0.1）
- PriceSpike（价格异动）
- VolumeSpike（量能异动）
- SectorResonance（板块共振）
- HotSectorUpdate（热门板块变更）

## 第一阶段模块边界
- Connector：快照与日线数据拉取
- Storage：事件与任务运行记录
- Scheduler：交易时段轮询 + 晨报定时
- Notifier：企业微信 webhook
- Report：晨报 Markdown

## 出口（可验证）
- `GET /health` 返回 ok
- 早间固定时间可推送一条晨报（即使内容为空也能发送）
- `task_runs` 记录每次任务执行与结果
